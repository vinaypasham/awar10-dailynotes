/dev	: Device files : Contains file the represent hardware devices i.e; hard disks.. 

/etc	: Configuration files : Stores system wide config files (i.e; /etc/passwd, /etc/groups)

/var	: Variable files : Contains data that changes frequently (logs, database, documenttoot path)

/usr	: user programs : contains user level programs and libs.

/sbin	: system binaries : System admin commands (ifconfig, iptables) - Only root/superuser can execute

/lib & lin64	: similar to windows .dll files : Essential shared library files.. 
/proc	: process inforamtion (cpu / RAM / processes)
/opt	: optional software
/mnt	: mount points
/media	: removable media

1. RPM (Redhat Package Manager) : Does not resolve dependenciies automatically. 

rpm -ivh package.rpm

2. YUM (Yellodog Updater Modified) : its a wrapper around rpm, that can automatically resolve dependencies and retrive packages from repositories (repo).

yum install httpd
yum install nginx

3. DNF (Dandified YUM) : All new linux distro are already using/switching to dnf as it works better for dependencies resolution.

dnf install httpd
dnf install nginx

dnf search <pkg-name>
dnf info <pkg-name>
dnf list installed 

dnf remove <pkg-name>

Repo Config : /etc/yum.repos.d (or) /etc/dnf/dnf.conf

------

Option 1 : Provide sudo permisisons to the user, so he can use "sudo" to install the packages. ec2-user by defaultly part of wheel, so he gets sudo permisisons automatically. 

sudo dnf install httpd

Option 2 : Switch to root user and perform the installations. : sudo su -

sudo su		: It wont fully load root users environment variables. (.bash_profile, .bashrc)
sudo su -	: Switch to root with login shell and it lods root;s environment variables. Simulates a fresh root login (Best practice for admin related activities)
sudo -i		: Same as (sudo su -), loads root user env var and moves to /root path.
